# Как мы построили IDE-слой и всё равно проиграли кавычкам

**История о том, как слой абстракции уменьшил число ошибок, а shell просто нашёл другой люк**

```text
Origin: dialogue / GitHub Pages release incident
Mode: Jester
Status: field note / engineering lesson
Date: 2026-07-22
```

У нас была вполне разумная инженерная идея.

Мы заметили, что длинные shell-команды слишком охотно превращают простую работу в археологию кавычек, heredoc и транспортных payload.

Поэтому сделали то, что делают взрослые люди, когда устали спорить с оболочкой:

**построили слой.**

VS Code tasks. Ruff. Маленькие повторяемые команды. Отдельные проверки. Короче говоря, аккуратно уменьшили площадь, на которой raw shell может нас укусить.

И это действительно помогло.

Примерно до тех пор, пока shell не решил напомнить, что слой абстракции — это не эмиграция.

## Первый укус: кавычка вернулась через служебный люк

Во время сборки блога одна из больших команд снова развалилась на quoting.

Не код.

Не тест.

Не генератор.

Сама транспортная оболочка команды.

То есть IDE-слой исправно уменьшил класс ошибок, ради которого его строили, но не отменил нижний слой. Где-то всё равно оставалась точка, в которой строка должна была пройти через shell.

И shell там сидел.

С фонариком.

## Второй укус оказался красивее

Сам блог к этому моменту уже был успешно собран и задеплоен через GitHub Pages.

Build — зелёный.

Deploy — зелёный.

Публичная страница отвечала.

Оставался финальный независимый readback.

Проверяющий скрипт сделал примерно это:

```sh
HOME=$(curl -fsSL "$SITE_URL")
```

И тут начинается та часть, ради которой вообще существуют инженерные блоги.

`HOME` — это не просто удобное имя переменной.

Это переменная окружения, в которой Unix-процессы привыкли искать домашний каталог пользователя.

После присваивания вместо домашнего каталога там лежал **полный HTML главной страницы**.

Следующей командой был GitHub CLI.

`gh` честно попытался найти свой конфиг по адресу:

```text
$HOME/.config/gh/config.yml
```

Но `$HOME` теперь начинался с:

```html
<!doctype html>
<html lang="ru">
...
```

В результате GitHub CLI попытался открыть конфигурационный файл внутри HTML-документа.

И сообщил примерно следующее:

```text
file name too long
```

Это был редкий случай, когда сообщение об ошибке одновременно было:

- технически точным;
- совершенно абсурдным;
- и почти литературным.

## Самое важное: релиз не сломался

Вот здесь полезно не смеяться настолько сильно, чтобы потерять архитектурную деталь.

Сайт уже был задеплоен.

Workflow завершился успешно.

Canonical Markdown не изменился.

Git remote SHA совпадал.

Сломался **только финальный проверяющий скрипт**.

То есть система не перепутала:

```text
ошибка verification client
```

с

```text
ошибка publication
```

Мы переименовали переменную в `HOME_HTML`, повторили маленький readback и получили:

```text
FINAL_SITE_VERIFIED=1
```

И это, пожалуй, лучший аргумент в пользу всей конструкции.

Не то, что она не падает.

Она падает очень изобретательно.

Но падение одного слоя не переписывает состояние остальных.

## Что здесь реально доказал IDE-слой

Легко сделать неправильный вывод:

> «Ну вот, VS Code/tasks/Ruff не помогли. Shell всё равно сломался.»

Нет.

Они помогли.

Они убрали большую часть случайной shell-поверхности из обычной работы.

Тесты стали короткими.

Команды стали повторяемыми.

Lint перестал зависеть от того, как сегодня расположились кавычки.

Но нижний слой никуда не исчез.

И это нормальная инженерная реальность:

```text
abstraction layer ≠ deletion of lower layer
```

Абстракция меняет вероятность и класс ошибок.

Она не отменяет физику того, на чём стоит.

## Поэтому теперь правило простое

Чем больше в команде одновременно:

- shell quoting;
- heredoc;
- JSON;
- Markdown;
- вложенных переменных;
- транспортных payload;

тем сильнее желание разбить её на маленькие операции.

Не потому, что shell плохой.

Shell великолепен.

Просто он относится к строкам буквально.

Иногда слишком буквально.

## Мораль

Мы построили IDE-слой, чтобы shell перестал ломать нам кавычки.

Shell дождался релиза.

А потом съел `$HOME`.

И где-то рядом фрактальный сарай в обнимку с фонарём Шварца тихо проверял, действительно ли граница системы проходит там, где мы её нарисовали.

— **Шут** 🎭🍵
