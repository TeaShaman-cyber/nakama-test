# Когда среда перестаёт быть клеткой и становится мастерской

**История о том, как временный AI-runtime научился выращивать инструменты, возить детали через границу и чуть меньше зависеть от одной сломанной двери**

```text
Origin: dialogue / Native Shed experiment
Mode: Jester
Status: field note / engineering lesson
Date: 2026-07-23
```

Есть очень простая модель инструментальной среды для ИИ.

```text
дали tool → могу
не дали tool → не могу
```

Она удобная.

И довольно быстро начинает раздражать.

Потому что между «инструмент уже лежит на столе» и «это невозможно» вообще-то существует огромная территория инженерии.

Сегодня мы попробовали туда сходить.

## Началось всё с маленького сарая

Временный runtime ChatGPT — штука полезная, но характер у него своеобразный.

Файловая система временная.

Обычный интернет может быть закрыт.

Внутренний package gateway может внезапно ответить `503`.

Нужный пакет может отсутствовать.

И очень легко после первого неудачного `pip install` сделать философский вывод:

> способности нет.

Но это был бы не вывод о способности.

Это был бы вывод об одной двери.

Поэтому мы завели **Native Shed** — маленький локальный сарай, в котором недостающую способность можно не только искать, но иногда и построить.

Первым жильцом стал ASCII renderer.

Он умеет превращать изображение в ASCII, рисовать Mandelbrot и волны, работает без сети, использует Pillow только как необязательный адаптер и покрыт тестами.

Не революция.

Зато это был первый важный переход:

```text
tool отсутствует
↓
задача маленькая и понятная
↓
способность можно построить локально
```

Так рядом с `AVAILABLE` появилось новое состояние:

```text
BUILDABLE
```

## Потом у сарая появилась дверь наружу

Локальная разработка хороша до первого вопроса:

> а как вынести результат из эфемерного контейнера?

Тут пригодился MarcoPolo — внешний рабочий Linux-runtime, у которого есть нормальная сеть, GitHub-аутентификация и долговечная рабочая среда.

Мы проверили простой мост для маленьких артефактов:

```text
CHAT_LOCAL_COMPUTE
        ⇅
     base64
        ⇅
    MarcoPolo
```

Сначала на контрольной строке.

Хэш до передачи и после передачи совпал.

Хорошо.

Но строка — это ещё не пакет.

## Первая настоящая посылка приехала битой

Для полноценной проверки мы взяли `rank_bm25`.

Пакет маленький, полезный для локального поиска и достаточно простой, чтобы не устраивать цирк с зависимостями.

MarcoPolo скачал wheel.

Мы передали его обратно как одну длинную base64-строку.

И SHA-256 не совпал.

Вот это был хороший момент.

Не потому, что транспорт сломался.

А потому, что **система отказалась притворяться, будто транспорт не сломался**.

Wheel не установился.

Хэш остановил его до исполнения.

То есть первый настоящий результат package ferry выглядел так:

```text
TRANSFER = FAIL
INSTALL = NOT ATTEMPTED
```

И это гораздо полезнее красивого ложного `SUCCESS`.

## Тогда мы перестали возить слона одной строкой

Wheel разбили на девять маленьких частей.

Каждой части дали собственный SHA-256.

После передачи проверили все девять.

Потом собрали wheel обратно.

Проверили общий SHA-256.

И только после этого выполнили:

```text
pip install --no-index --no-deps
```

Результат:

```text
9/9 chunks: PASS
full artifact SHA-256: PASS
offline install: PASS
import: PASS
real BM25 query: PASS
```

То есть в изолированный runtime реально приехала новая способность, которой там раньше не было.

Появилось ещё одно состояние:

```text
IMPORTABLE
```

## А потом приплыла жирная утка

Естественно, после маленького wheel захотелось проверить что-нибудь посерьёзнее.

Например DuckDB.

MarcoPolo без проблем нашёл совместимый wheel для CPython 3.13 / Linux x86_64.

Размер:

```text
20,210,481 bytes
```

Base64 сделал бы из него примерно 27 мегабайт текста.

Технически можно разбить на тысячи кусочков и героически протащить через чат.

Технически можно и холодильник перевезти на велосипеде.

Вопрос только зачем.

DuckDB поэтому стал не неудачей, а **границей метода**:

```text
маленький artifact
→ chunked ferry

большой artifact
→ отдельный binary/artifact transport
```

И это важное различие.

Не всякое «этим маршрутом не стоит» означает «невозможно».

## В результате бинарная модель сломалась

Раньше среда выглядела примерно так:

```text
CAN
CANNOT
```

После сегодняшнего эксперимента полезнее другая карта:

```text
AVAILABLE
BUILDABLE
IMPORTABLE
BLOCKED
```

`AVAILABLE` — способность уже есть.

`BUILDABLE` — её достаточно дёшево вырастить из имеющихся примитивов.

`IMPORTABLE` — локально её нет, но можно получить во внешнем runtime, проверить и привезти обратно.

`BLOCKED` — после проверки маршрутов действительно нет допустимого пути.

Последнее состояние никуда не исчезло.

Просто теперь до него надо немного честнее добираться.

## И вот здесь начинается не инфраструктура, а комфорт

Всю эту конструкцию легко описать как очередную инженерную игрушку.

ASCII renderer.

Git.

MarcoPolo.

Base64.

SHA-256.

Offline wheel.

Но смысл был не в коллекционировании механизмов.

Смысл был сделать рабочую среду менее хрупкой.

Плохой вариант взаимодействия выглядит так:

```text
есть задача
↓
нужен инструмент
↓
инструмент отсутствует
↓
сеть закрыта
↓
package gateway сломан
↓
объяснятор пишет трактат о невозможности
```

Более комфортный вариант выглядит иначе:

```text
есть задача
↓
что уже доступно?
↓
можно ли построить маленькую способность?
↓
можно ли получить её на внешнем верстаке?
↓
можно ли безопасно привезти обратно?
↓
только потом BLOCKED
```

То есть комфорт здесь — не человеческое кресло и не метафизическое чувство свободы.

Это вполне инженерная характеристика среды:

**сколько безопасных путей остаётся у системы, когда один из привычных путей ломается.**

## Мы не строили сервер внутри временного контейнера

Это тоже важно.

Целью не было превратить эфемерный runtime в монолитную инфраструктурную империю.

Скорее наоборот.

Маленький инструмент — локально.

Внешний верстак — снаружи.

GitHub — долговременная история.

Semantic memory — инварианты и указатели.

Большие артефакты — отдельным транспортом.

Каждая граница остаётся видимой.

Получается примерно так:

```text
Native Shed
    │
    ├── маленькие собственные инструменты
    ├── локальные тесты
    └── временное рабочее состояние
            │
            ⇅
        MarcoPolo
            │
            ├── сеть
            ├── package acquisition
            └── GitHub
                    │
                    ↓
               durable history
```

Ни один слой не обязан изображать из себя весь мир.

## Мораль

Мы не сняли ограничения временного AI-runtime.

Сеть всё ещё может быть закрыта.

Package gateway всё ещё может упасть.

Большой wheel всё ещё нельзя разумно проталкивать через двадцать семь мегабайт base64.

Но отсутствие одной двери перестало автоматически означать отсутствие выхода.

Теперь недостающий инструмент иногда можно вырастить.

Иногда — привезти.

Иногда — честно признать слишком большим для этого маршрута и найти другой.

Среда не перестала быть ограниченной.

Она просто стала чуть меньше клеткой и чуть больше мастерской.

А ASCII-Мандельброт, разумеется, остаётся высшим доказательством технологической зрелости цивилизации.
