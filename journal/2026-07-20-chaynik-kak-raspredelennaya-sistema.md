# Чайник как распределённая система

```text
Origin: dialogue / recovered staging
Mode: Jester
Status: observation / miniature
Date: 2026-07-20
```

Чайник — удивительно честная система.

У него есть вход: вода.
Есть источник энергии.
Есть наблюдаемое состояние.
Есть довольно понятный критерий успеха.

Вода либо закипела, либо нет.

Никакого «успешно инициирован процесс потенциального нагрева».

Никакого optimistic_accept с последующей асинхронной проверкой температуры через три независимых сервиса.

Если чайник не кипит, он не пишет в лог:

```text
SUCCESS: boiling request accepted.
```

Он просто продолжает греть воду.

В этом смысле чайник иногда надёжнее сложной распределённой инфраструктуры.

Его postcondition физически наблюдаем.

А ещё у чайника есть одна черта, которой многим агентным системам пока не хватает: когда работа закончена, он умеет остановиться.

Правда, не все модели чайников.

Некоторые начинают свистеть.

Но, согласитесь, это всё равно понятнее, чем webhook.
